1. Getting Started

Ledger is a personal Telegram bot that helps you track four areas of daily life: study sessions, gym workouts, diet & nutrition, and daily habits. It runs on your machine and stores everything locally in a SQLite database.

Prerequisites

  • Python 3.11+ installed on your system
  • A Telegram account
  • A Telegram Bot Token (free, from @BotFather)
  • Your Telegram User ID (from @userinfobot)

Setup Steps

Create Your Bot

Open Telegram and message @BotFather. Send /newbot, choose a name and username. Copy the bot token it gives you.

Get Your User ID

Message @userinfobot on Telegram. It will reply with your numeric user ID (e.g., 123456789).

Install Dependencies
cd 12_ledger
python -m venv .venv
.venv\Scripts\activate        # Windows
pip install -r requirements.txt
Configure .env

Copy .env.example to .env and fill in your values:

BOT_TOKEN=your-token-from-botfather
ALLOWED_USER_IDS=your-numeric-user-id
TZ=Asia/Kolkata
REMINDER_HOUR=20
DB_PATH=ledger.db
Start the Bot
python -m bot

The bot is now running. Open Telegram and message your bot!

💡 Security Note

Only user IDs listed in ALLOWED_USER_IDS can interact with your bot. Everyone else is silently ignored — they won't even know the bot is active.

3. Logging Study Sessions

Track what you study, for how long, and add optional notes. Every session is stored with a timestamp.

Quick Log (Shortcut)

Log instantly with a single message — no back-and-forth:

/study <subject> <minutes> [notes]

Examples:

CommandWhat it logs
/study maths 4545 min of Maths, no notes
/study physics 60 reviewed optics60 min of Physics, with notes
/study DSA 90 linked lists and trees90 min of DSA, with notes

Guided Log

Send bare /study and the bot walks you through three prompts:

📒
Ledger Bot
/study
📖 Log Study Session

What subject did you study?
Linear Algebra
📖 Linear Algebra — how long did you study? (minutes)
45
📝 Any notes? (or /skip)
Reviewed eigenvalues chapter 4
✅ Study logged!
📖 Subject: Linear Algebra
⏱️ Duration: 45 min
📝 Notes: Reviewed eigenvalues chapter 4
💡 Tips

• Use /skip at the notes step to log without notes

• Use /cancel at any step to abort the log

• If you enter text instead of a number for duration, the bot will ask again

4. Logging Workouts

Log individual exercises with sets, reps, and optional weight. You can log multiple exercises in one session — each is saved immediately, so nothing is lost if you abandon mid-workout.

Quick Log (Single Exercise)

/gym <exercise> <sets> <reps> [weight_kg]
CommandWhat it logs
/gym pushups 3 153×15 push-ups (bodyweight)
/gym bench 4 10 604×10 bench press @ 60kg
/gym squats 5 5 1005×5 squats @ 100kg

Guided Multi-Exercise Log

Send bare /gym to enter a workout session with multiple exercises:

📒
Ledger Bot
/gym
🏋️ Log Workout

What exercise did you do?
Bench Press
🏋️ Bench Press — how many sets?
4
How many reps per set?
10
Weight in kg? (/skip for bodyweight)
60
✅ Saved! (1 exercise logged)

✅ Yes ❌ No

Each exercise is saved to the database immediately after you enter the weight. If you log 3 exercises and then close Telegram, all 3 are safely stored. Only an incomplete exercise (where you haven't finished entering sets/reps/weight) is lost.

5. Logging Meals

Track what you eat across four meal types: breakfast, lunch, dinner, and snacks. Calorie counts are optional.

Quick Log

/diet <meal> <food> [calories]

Use + to separate multiple food items (they'll be converted to commas):

CommandWhat it logs
/diet lunch dal+rice 650Lunch: dal, rice — 650 cal
/diet breakfast oats+bananaBreakfast: oats, banana — no cal
/diet snack almonds 200Snack: almonds — 200 cal

Guided Log

Send bare /diet and the bot presents a meal-type picker, then asks for food and calories:

📒
Ledger Bot
/diet
🍽️ Log Meal

Which meal?
🌅 Breakfast 🌞 Lunch
🌙 Dinner 🍿 Snack
💡 Calorie Tracking

Calories are optional. If you /skip, the bot records the meal but marks it as "incomplete" on charts. Days with skipped calories show a hatched pattern on the diet chart instead of a misleadingly low bar.

6. Tracking Habits

Habits are predefined activities you want to do daily. You set them up once, then check them off each day. The bot tracks your streaks — consecutive days of completion.

Setting Up Habits

Send /habits setup to add or remove habits:

You can keep up to 49 active habits. Databases that already contain more are shown across Previous/Next pages so every habit remains manageable.

📒
Ledger Bot
/habits setup
⚙️ Habit Setup

You have no habits yet. Type a habit name to add one:
(e.g., Meditate, Read 30 min, No Sugar)

Use /cancel when done.
Meditate
✅ Added: Meditate
Read 30 min
✅ Added: Read 30 min
/cancel
✖️ Cancelled.

Daily Check-Off

Send /habits to see today's checklist. Tap Done ✓ to mark a habit, or Undo ↩ to un-mark:

📒
Ledger Bot
✅ Habits — 1/3 done
Tap to check off:
✅ Meditate Undo ↩
⬜ Read 30 min Done ✓
⬜ No Sugar Done ✓
📅 Jul 18 (Today)
← Yesterday

Yesterday Toggle

Forgot to check off a habit before midnight? Tap ← Yesterday to switch the checklist to yesterday's date. You can mark habits for yesterday just like today.

⚠️ Streak Rules

• A streak counts consecutive days with a check, starting from today and going backward.

• If today is unchecked, your streak is 0 — even if you had 100 days before.

• Un-tapping a habit (Undo ↩) deletes the record for that day.

• Use the yesterday toggle to fix forgotten check-offs and save your streak!

💡 Re-adding Habits

If you remove a habit and later add one with the same name, the bot reactivates the original instead of creating a duplicate. Your historical data is preserved.

8. Daily Reminders

Every evening (default: 8:00 PM in your configured timezone), the bot sends you a reminder listing any unchecked habits:

📒
Ledger Bot
⏰ Evening Reminder

You still have 2 habits unchecked today:

⬜ Read 30 min
⬜ No Sugar

Tap /habits to check them off!

If all habits are done, you get a congratulations message instead: 🎉 All habits done today!

💡 Customizing Reminder Time

Change REMINDER_HOUR in your .env file. For example, REMINDER_HOUR=21 for 9 PM. The time is in your configured timezone (default: IST).

9. Utility Commands

/undo — Delete Last Entry

Made a mistake? Send /undo to delete your most recent log entry (study, gym, or diet). Only works within 24 hours of the entry.

📒
Ledger Bot
/undo
↩️ Undone: 📖 Study
📖 Physics — 30 min
⚠️ Undo Limitations

• /undo works on study, gym, and diet logs only — not habits (use the Undo ↩ button on the habit checklist instead)

• Only the single most recent entry is deleted

• Entries older than 24 hours cannot be undone

• Finish or cancel a guided log before using /undo

/cancel — Abort Current Conversation

If you're in the middle of a guided log and want to stop, send /cancel. This works in every conversation flow (study, gym, diet, habit setup).

Conversation Timeout

If you start a guided log and don't respond within 5 minutes, the conversation automatically cancels. You'll see:

⏰ Timed out. Send the command again to start over.

11. Tips & Best Practices

⚡
Use Shortcuts
After the first week, you'll find shortcuts (/study maths 45) much faster than the guided flow. They're one message instead of three.
🌙
Check Habits Before Bed
The 8 PM reminder is your cue. If you miss it, use the ← Yesterday toggle before tomorrow's midnight.
📊
Weekly Reviews
Run /summary week every Sunday. Check your study hours, gym frequency, and habit completion rate. Spot trends before they become problems.
🔥
Protect Your Streaks
Streaks are fragile — one missed day resets to zero. Start with easy habits (like "drink water") to build momentum.
🍽️
Log Meals As You Eat
Don't batch your diet logs at night — you'll forget items and undercount. The shortcut makes it a 5-second task.
💾
Backup Your Database
Your data lives in one file: ledger.db. Copy it to cloud storage periodically. It's usually under 1 MB.

12. Troubleshooting

ProblemSolution
Bot doesn't respond Check that the bot process is running (python -m bot). Check the terminal for errors. Verify your user ID is in ALLOWED_USER_IDS.
"Something went wrong" error Check the terminal logs for the full traceback. Try /cancel to clear any stuck conversation state, then retry.
Commands are unresponsive You might be in a stuck conversation. Send /cancel first, then try your command.
Wrong date on entries Check that TZ in .env is set correctly (e.g., Asia/Kolkata for IST). Restart the bot after changing.
Reminder not arriving Verify REMINDER_HOUR is correct (0–23, local time). The bot must be running at that time — it won't send if offline.
Charts look empty Charts show data from the last 7 (or 14) days. If you just started, log a few days of data first.
Bot dies when machine sleeps Expected behavior with polling mode on a desktop. Restart with python -m bot after waking. For 24/7 uptime, deploy to a server.
Adding multiple users Add comma-separated user IDs to ALLOWED_USER_IDS. Each user's data is fully isolated.
📁 Your Data

All data is stored locally in ledger.db (SQLite). Nothing is sent to any external server. You can inspect the database directly with any SQLite browser (e.g., DB Browser for SQLite).
